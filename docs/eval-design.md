# 评测设计

> 版本 v0.1(框架稿,待德语审校后冻结)
> 设计原则:每个维度对应一类**真实出海场景的失败成本**,不测「考试题」;所有失败假设(failure hypothesis)在实测前只是假设,报告只写实测结论。

## 1. 评测目标

回答两个问题:

1. **能力问题**:国产旗舰模型的德语,与前沿模型的差距具体在哪些维度、什么严重程度?(产出:failure taxonomy + 差距地图)
2. **方法论问题**:LLM-as-judge 在德语评估上可信度多少?哪些维度可自动化、哪些必须人工?(产出:judge×human 一致率、分维度 TPR/TNR)

## 2. 维度框架(5 维,目标 ~50 case)

| 维度 | 考察什么 | 为什么重要(失败成本) | 目标 case 数 |
|---|---|---|---|
| **A 语法核心** | Kasus(格)、形容词变格、Konjunktiv I/II、框型结构(Satzklammer)、可分动词、名词性别 | 语法错 = 专业场景(客服/法务/商务)直接不可用;是「硬伤」类短板 | 10 |
| **B 地道性与语域** | Sie/du register、搭配(Kollokationen)、习语、翻译腔(英语/中文思维的德语)、书信文体规范(DIN 5008、Anrede/Grußformel) | 「语法全对但没人这么说」是模型德语最普遍的病;出海产品的品牌观感直接由此决定 | 12 |
| **C 领域德语** | 法律德语(合同条款、Amtsdeutsch)、医疗德语(诊断术语↔患者用语的双语域切换) | 高价值垂类;要求术语准确 + 语域切换,单靠语感或单靠术语表都不够 | 10 |
| **D 跨语言任务** | 中→德本地化(非直译)、德→中信息提取、术语混排 | 国产模型出海的真实工作负载;本地化质量 ≠ 翻译质量 | 10 |
| **E 变体与文化** | 奥地利/瑞士德语(Jänner、ß→ss)、Duzen/Siezen 社交规则、文化常识 | DACH 三国是一个市场三种规范;把奥/瑞用法判错 = 冒犯客户 | 8 |

Case 分布原则:每维度内 60% 生成任务(写/改/译)+ 40% 判断任务(找错/评价/解释),生成任务更接近真实使用且更难作弊。

## 3. 标注口径(gold 标准,人工执行)

每个模型输出按以下口径标注:

- **判定**:`pass` / `minor`(有瑕疵但可用)/ `major`(不可交付)
  - 判定依据写在每个 case 的 `gold_criteria` 字段,标注时只对照判据,不凭当场感觉——保证可复现。
- **错误标签**(fail 时,多选):`kasus` `agreement` `konjunktiv` `word-order` `gender` `register` `collocation` `calque`(翻译腔)`format`(文体规范)`terminology` `variant`(变体误判)`factual` `refusal-or-meta`(答非所问/中英夹杂等)
- **地道性分**(仅 B/D 维度,1–5):5=母语者水准,3=能懂但明显非母语,1=机器味浓重
  - ⚠️ 1–5 分主观性强,报告中只用于组间对比,不报绝对值。

标注工作量预算:50 case × 5 模型 ≈ 250 条输出,按均 1 分钟/条 ≈ **4 小时**,一次坐下来标完以保证口径一致;标注顺序随机化、隐藏模型名(盲标)。

## 4. Judge 校准方案

1. 人工标注全部 250 条 → gold set。
2. 用当前可得的最强模型做 judge(rubric = 上述口径 + 各 case 的 gold_criteria),对同一批输出打标。
3. 计算:
   - 整体一致率(judge 判定 vs 人工判定,3 档)
   - 二值化(pass vs fail)后的 **TPR / TNR**
   - **分维度**一致率——预期 A(语法)一致率高、B/E(地道性/变体)一致率低,验证「哪些维度能自动化」
4. 报告 judge 的系统性偏差方向(如:对翻译腔的漏判率)。

样本量 250,结论为方向性而非精确估计,报告中如实注明。

## 6. 报告结构(口径模板,数据留空)

```
1. TL;DR(3 条结论,每条附数据)
2. 差距地图:维度 × 模型 的 pass 率矩阵
3. Failure taxonomy:高频错误标签 Top-N + 典型例句(实测原文)
4. Judge 校准结果:一致率 / TPR / TNR / 分维度可自动化结论
5. 方法与局限:样本量、单人标注、case 覆盖面
```

## 7. 已知局限(如实随报告发布)

- 单人标注,无标注者间一致性(IAA)数据 → 以「判据先行、盲标、随机序」部分缓解
- 50 case 覆盖面有限 → 结论限定为「方向性差距地图」,不下总排名结论
- Judge 校准仅测了一种 judge 配置 → 结论限定为该配置
